# project-planning

## Purpose
Use this skill to break the repository work into small academic phases, keeping the scope simple and aligned with the approved stack.

## Instructions
- Prefer phased delivery over large rewrites.
- Keep plans actionable and short.
- Separate repository setup, backend, NLP, rules, frontend, and validation.
- Reject scope creep that adds unsupported technologies.

## Expected Output
- Updated roadmap in `PLANS.md`
- Small, traceable implementation steps
