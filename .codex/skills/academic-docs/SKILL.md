# academic-docs

## Purpose
Use this skill to keep project documentation aligned with an academic, modular, and easy-to-explain implementation.

## Instructions
- Write concise explanations.
- Prefer clear architecture and methodology notes over marketing language.
- Document project limits, assumptions, and future work.
- Keep README and docs synchronized with the actual implementation state.

## Expected Output
- Updated README and `docs/`
- Short explanations suitable for coursework or demos
